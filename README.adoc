= Pulpogato Demo

Demo Spring Boot app for https://github.com/pulpogato/pulpogato[Pulpogato], a generated Java client for the GitHub API.

This project shows two things:

. How to resolve Pulpogato artifacts from Maven Central, GitHub Packages, or JitPack
. How to handle typed GitHub webhooks (`ping` and `push`) with Pulpogato in Spring MVC

== Prerequisites

* Java 21
* The Gradle Wrapper in this repo (`./gradlew`)

For GitHub Packages only, set:

* `GITHUB_ACTOR` — your GitHub username
* `GITHUB_TOKEN` — a token with `read:packages`

== Dependency management

Look at `build.gradle.kts` for how the Maven group, repositories, and credentials change per source.
Defaults live in `gradle.properties` (`repo=central`, `ghVersion=fpt`, `pulpogatoVersion`, and so on).

|===
| `repo` value | Source

| `central` (default)
| Maven Central

| `github`
| GitHub Packages (`maven.pkg.github.com/pulpogato/pulpogato`)

| `jitpack`
| JitPack
|===

Build against all three (GitHub Packages is skipped if `GITHUB_ACTOR` is unset):

[source,bash]
----
./scripts/build-all.sh
----

Or build one source at a time:

[source,bash]
----
./gradlew build -Prepo=central
./gradlew build -Prepo=jitpack
./gradlew build -Prepo=github \
  -Pgpr.user="$GITHUB_ACTOR" \
  -Pgpr.key="$GITHUB_TOKEN"
----

== Running the app

The app exposes `POST /github-webhook`.
`HooksController` implements Pulpogato's `PingWebhooks` and `PushWebhooks`, so incoming payloads are bound to typed models such as `WebhookPing` and `WebhookPush`.

Start it with:

[source,bash]
----
./gradlew bootRun
----

== Sending a test webhook

With the app running, replay a sample GitHub `ping` delivery from `src/test/resources/ping.http`:

[source,bash]
----
./scripts/send-request.sh
----

Optional arguments: base URL (default `http://localhost:8080`) and path to an `.http` file:

[source,bash]
----
./scripts/send-request.sh http://localhost:8080 src/test/resources/ping.http
----
